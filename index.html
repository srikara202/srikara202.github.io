Llama-2-7B → English-to-SQL (LoRA fine-tune + Colab Streamlit demo)

LLM fine-tuning · PEFT · quantization
Fine-tuned LLM 2026

I fine-tuned Llama-2-7B to generate SQL from a plain-English question plus a table schema (DDL). The goal was a practical end-to-end workflow that runs on a single Colab GPU using parameter-efficient training. The demo UI runs inside Colab via a tunnel so you can type an instruction + schema and get SQL back.

Repo Colab notebook
Story (Problem → Solution → Result)
Problem: Writing SQL is a common bottleneck. Users often have a question + schema but need help translating it into correct SQL.
Solution: Fine-tuned Llama-2-7B with instruction-style prompting using LoRA adapters + 4-bit loading (NF4) to fit training in Colab.
Result: A working demo that generates SQL with controllable decoding (temperature, top-p, top-k). Trained on a 5,000-example subset for a quick iteration loop.
Aha insight: Exact string match is misleading for SQL (many answers can be correct). Execution-based evaluation is the right way to measure correctness.
V2 improvement:
  • Add execution-based evaluation (run SQL on a test DB) + dialect controls (SQLite/Postgres/MySQL).
  • Handle long schemas with schema pruning / retrieval (and better context-window management).

GPT-2 Foundational Model Built From Scratch (PyTorch) + distilgpt2 weight loading

Transformers from scratch · tokenization · decoding
Paper reimplementation 2026

I implemented GPT-2 end-to-end in PyTorch (token + positional embeddings, causal self-attention, transformer blocks, LM head), built a toy training pipeline, and then loaded HuggingFace distilgpt2 weights into my own architecture. I sanity-checked correctness via near-identical logits and matching argmax tokens.

Repo Live demo
Story (Problem → Solution → Result)
Problem: I wanted a first-principles understanding of transformer internals (attention math, masking, shapes, and weight mapping).
Solution: Implemented the full GPT-2 stack in PyTorch and wrote utilities for tokenization, windowed training pairs, and autoregressive generation (temperature + top-k).
Result: Loaded distilgpt2 (82M) weights and validated outputs: max abs diff ≈ 3.8×10⁻⁵ with matching argmax tokens.
Aha insight: “Correctness” isn’t just that it runs, tiny implementation details (masking, layernorm placement, shape conventions) show up immediately in logits.
V2 improvement:
  • Add better sampling defaults + repetition penalties (toy-trained models can be incoherent).
  • Expand training + evaluation (perplexity) on a standard dataset with stronger checkpoints.

Diabetes Risk Prediction System, recall-focused clinical ML + Streamlit app

Healthcare ML · threshold optimization · responsible UI
Original build 2025

A complete ML web app for diabetes risk assessment where the optimization target is clinical: prioritize recall to reduce missed cases, and ship the result with guardrails and clear disclaimers.

Repo Live demo
Story (Problem → Solution → Result)
Problem: In screening-like settings, false negatives can be more harmful than false positives.
Solution: Trained a baseline classifier and tuned decision thresholds to explicitly trade precision for recall in a patient-safety-oriented way.
Result: Recall improved from 51.9% → 70.4% (≈35% relative improvement), and false negatives reduced from 48% → 30%.
Aha insight: “Best AUC” doesn’t automatically mean “best clinical behavior”, thresholding is where you align the model with real-world costs.
V2 improvement:
  • Add calibration (reliability curves) so predicted risks align with observed frequencies.
  • Integrate model cards + more explicit uncertainty / limitations messaging.
